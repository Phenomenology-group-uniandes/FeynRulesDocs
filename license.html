Overview & Workflow

Last updated: Nov 12th, 2020

Overview

FeynRules is the initial step for the study of a novel extension of the SM. It addresses the implementation of particle physics models, which are given in the form of a list of fields, parameters and a Lagrangian. FeynRules computes the underlying Feynman rules needed to build and implement the model into simulation software.

The main problem to develop a computer based theoretical model was that each of the programs for event generation required the model to be written in their own syntax. Besides, implementing a new model often required to modify the event generator underlying code. This means that in some of these programs it was required the hand coding of each vertex. Consequently, the communication between experimental and theoretical physicists was tedious and the models were very error prone.

To overcome this problem three software were developed, one of them FeynRules. This program aims to provide theorists a quick way to implement their new models into simulation packages using an unified syntax. The approach is based on the implementation of a Lagrangian into a file written in Mathematica language. Then, the software computes the vertices from the Lagrangian and store them into a universal file known as UFO, thus saving theorists time and errors.

Useful Info:

The Lagrangian has several parameters and fields, this elements must be defined in the model as objects with certain attributes as group symmetries, dimension and the type of parameter or field and it allows to define models at LO up to NLO.

WorkFlow

In the LHC occurs around 1 billion collision per second, at which BSM decais could occur as an intermediate or a final state. Given the huge amount of data collected in the LHC, the correct detection of a DM decay, within several noise sources, is rather a complex task. One way to approach this search of DM is to simulate several events for promising decays processes and compare them to similar backgrounds in energy and momentum. This comparison allows to perform an identification of key features that separate the signal from the background or to discard the event for promising detection.

To perform such analysis first and foremost a computational model is developed through a configuration file that recreates the behavior described by the Lagrangian. The most universal format for a particle physics model is known as Universal FeynRules Output (UFO). The UFO model format is independent of any individual Feynman diagram calculator and it incorporates the full information contained in the model. This model is compatible with multiple frameworks such as MadGraph 5, MadAnalysis 5, Aloha, GoSam and Herwig++. To generate this file one can use a Mathematica-based package called FeynRules, and it permits to write the model in a high level language as it is Mathematica.

Although FeynRules provides a series of build-in functions that allow superficial validation of the model, other tools are needed to completely verify it. In this work, a Mathematica package known as FeynArts is used for the generation and visualization of Feynman diagrams and amplitudes. Then, the model was being refined from several modifications in FeynRules that were validated in FeynArts, through the comparison of analytical results. Then with the final model, MadGraph 5 can generate all the parameters needed to evolve numerous generated events.

To write the model, it is required to start by entering the details of the new model ina form that can be parsed by FeynRules. Then specify a set of special variables and macros such as: model information, index definitions, model parameters (scalar ortensorial), particle classes, gauge groups, model restrictions, between others3. Next,with all the parameters defined, the full Lagrangian that describes the interactionsin the model can be written. The model can be defined in the interface line byline, but the most practical way is to save the model into a .fr text file. The full implementation of our model is described in the Model Implementation Section.
