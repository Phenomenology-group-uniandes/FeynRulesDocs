Connecting to MadGraph

Last updated: Nov 12th, 2020

MadGraph

Once the complete model is defined, the user is able to compute the UFO as is shown in the Code.. First, the code defines the location of the feynrules package, loads the package and it sets the Working Directory in which the model file heavyN.fr is located. Secondly, the program loads the standard model file sm.fr, the SM extension heavyN.fr, the restrictions for the CKM matrix and sets the feynman gauge to avoid the non-physical Goldstone modes. After loading the model and restrictions, the user can check several properties by build-in functions of the FeynRules packages. As an example the Code checks the Hermicity of the Lagrangian and computes the mass spectrum of each massive physical particle in the model. Finally, the code computes the Feynman rules for the Lagrangian with the function FeynmanRules[ ], and from these rules computes the UFO file with the command WriteUFO[]

Key Specs

The Universal Feynman Output (UFO) is a set of python files that completly define the model at each level, identifying also the particles parameters and matrix elements

MadGraph exportation example

                                $CurrentPath = Directory[]
                                $FeynRulesPath = SetDirectory["/\feynrules-current"];
                                << FeynRules`
                                SetDirectory["\feynrules-current/majoronLO/SM_HeavyN_FilesWithUFO"];
                                LoadModel["sm.fr", "heavyN.fr"];
                                LoadRestriction["diagonalCKM.rst", "massless.rst"];
                                FeynmanGauge = True;
                                CheckHermiticity[LFull]
                                CheckMassSpectrum[LFull]
                                FeynmanRules[LFull]
                                WriteUFO[LSM, LN]
